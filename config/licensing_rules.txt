

Generalized Rules:


Licensing rules:

1. Overall Active licenses:
   - Primary Table: SF_Account_LK_LOAD
   - Date Column: License_Expiry_Date__c
   - Category Column: Legal_Form__c 
   - SQL Query: SELECT COUNT(DISTINCT Id) AS ActiveLicense FROM SF_Account_LK_LOAD
                WHERE Current_License_Number__c IS NOT NULL
                AND Is_Expo_Commercial__c = FALSE
                AND Is_Expo_Concession__c = FALSE
                AND EXPO_Participants_Registration__c = FALSE
                AND Company_Deregistration_Date__c IS NULL
                AND Legal_Form__c IN ('DWC-Branch', 'DWC-LLC')
                AND Record_Type_Name__c <> 'Freelancer'
                AND Name <> 'International Trading (Test) 1'
                AND Current_License_Status__c IS NOT NULL
                AND License_Expiry_Date__c >= CURRENT_DATE

   - RULE : When the query is asking for the total number of active licenses across all accounts, without specifying a breakdown by country, department, or legal form, use this query:
            Provide only the total active license count satisfying all current-status filters.

2. Active Licenses by a Category (Group by Country/Department/Legal form):
   - Primary Table: SF_Account_LK_LOAD
   - Date Column: License_Expiry_Date__c
   - Category Column: CountryName_of_Origin__c (These are the Country Names)
                      Department__c (These are the Departments from where License are attained)
                      Legal_Form__c  ('DWC-Branch', 'DWC-LLC')
   - SQL Query: SELECT CountryName_of_Origin__c,COUNT(DISTINCT Current_License_Number__c) AS ActiveLicense FROM SF_Account_LK_LOAD
                WHERE YEAR(License_Expiry_Date__c) >= YEAR(CURRENT_DATE)
                AND Legal_Form__c IN ('DWC-Branch', 'DWC-LLC')
                AND EXPO_Participants_Registration__c = FALSE
                AND Is_Expo_Concession__c = FALSE
                AND Is_Expo_Commercial__c = FALSE
                group by CountryName_of_Origin__c

   - RULE : When the query requests active licenses grouped by any category such as Country, Department, or Legal Form, use this query:
            Provide a count grouped by the specified category, showing the number of active licenses in each group, using only records matching group conditions.

3. Active Licenses by ShareHolders Type
   - Primary Table: SF_Account_LK_LOAD
   - Date Column: License_Expiry_Date__c
   - SQL Query: SELECT S.Type_of_Shareholder__c,COUNT(DISTINCT S.Id) AS ActiveLicense FROM SF_Account_LK_LOAD A
                LEFT JOIN SF_ShareOwnership_LK_LOAD S on S.Company__c=A.Id
                WHERE YEAR(A.License_Expiry_Date__c) >= YEAR(CURRENT_DATE)
                AND A.Legal_Form__c IN ('DWC-Branch', 'DWC-LLC')
                AND A.EXPO_Participants_Registration__c = FALSE
                AND A.Is_Expo_Concession__c = FALSE
                AND A.Is_Expo_Commercial__c = FALSE
                group by S.Type_of_Shareholder__c

4. Expired Licenses:

5. Expected Revenue:
   - Primary Table: SF_LicensingRequest_LK_LOAD1

   - SQL Query: Select sum(RF.Total__c) as ExpectedRevenue from SF_LicensingRequest_LK_LOAD1 L

                AND L.Is_Test_Record__c= FALSE

   - Request_Type__c: 

6. License Renewals:
   - Primary Table: SF_Account_LK_LOAD
   - Date Column: SF_License_LK_LOAD.License_Renewed_Date__c
   - Category Column: Department__c
   - SQL Query: Select count(DISTINCT L.Id) from  SF_Account_LK_LOAD A
                LEFT JOIN SF_License_LK_LOAD L on L.Licensed_Company__c=A.Id
                where YEAR(L.License_Renewed_Date__c) >= YEAR(Current_date)
                AND L.License_Renewed_Date__c is not null
                AND A.EXPO_Participants_Registration__c=FALSE
                AND A.Is_Expo_Commercial__c=FALSE
                AND A.Is_Expo_Concession__c=FALSE 
                AND A.Record_Type_Name__c <> 'Freelancer'
                and L.License_Status__c='Renewal'

7. Freelancers Active License:

8. Freelancers Renewal License:

22. Total Licensing Team Task:

. Time filters (Fabric Lakehouse SQL - preferred, safe across year boundaries):
   - DEFAULT (no explicit time qualifier in user question): WHERE YEAR(Date column) = YEAR(current_date())
   - Last month:WHERE YEAR(Date column) = YEAR(current_date()) AND MONTH(Date column) = MONTH(current_date()) -1 
   - Last quarter:WHERE YEAR(Date column) = YEAR(current_date()) AND QUARTER(Date column) = QUARTER(current_date()) - 1
   - Last year:WHERE YEAR(Date column) = YEAR(current_date()) - 1
   - Year-to-date (YTD):WHERE YEAR(Date column) = YEAR(current_date()) AND Date column <= current_date()
   - Monthwise grouping (use these expressions  GROUP BY): GROUP BY YEAR(Date column), MONTH(Date column)
   - Quarterwise grouping: GROUP BY YEAR(Date column), QUARTER(Date column)
   - Yearwise grouping: GROUP BY YEAR(Date column)

. Always alias aggregated fields clearly (e.g., SUM(Amount__c) AS Revenue, COUNT(DISTINCT Id) AS Deals).

Notes for the LLM / SQL generator:
- Recognize common phrasings (Q2, last Q, current year, this year, last month, month-to-date) and map them to the patterns above.
- If the result is shown by category, include the overall sum/total at the end.
